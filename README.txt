			ОБЛІК З ПРОДАЖУ АВТОМОБІЛІВ



В цьому проекті знаходиться шість файлів які взаємодіють між собою:

1. create_bd.py

2. database.py

3. tables.py

4. CRUD.py

5. reports.py

6. additional_functions.py

7. main.py

8. .env


	ЗАПУСК БД З НУЛЯ:

1. Потрібно створити БД у PostgreSQL
2. Переконатись чи файл .env містить правильні параметри підключення :
	DB_USER=ваш користувач
	DB_PASSWORD=ваш пароль
	DB_HOST=localhost
	DB_PORT=5432
	DB_NAME=назва БД
3. Запустіть create_bd.py для створення таблиць
	після цього у вашій БД зявляться таблиці: employee, cars, sales 
4.Для взаємодії (додавання, видалення або зміни будь якої інформації) запустіть:
	main.py

	ОПИС ВСІХ ФАЙЛІВ:

1. create_bd.py
Створення таблиць які описані у tables.py


2. database.py
В цьому файлі створене все необхідне для підключення та взаємодії з БД


3. tables.py

Тут створені таблиці які будуть зберігати всю необхідну інформацію про: 	-	
	- працівників

	- автомобілів

	- продажі



4. CRUD.py

Центр взаємодії з Таблицями. Тут у нас відбуваються зміни такі як: 	-	-	- -	- додавання(працівника/авто/продажу) 

	- зміна будь якої інформації(працівника/авто) 

	- видалення(працівника/авто/) 

	- перегляд інформації(працівника/авто)



5. reports.py

В цьому файлі у нас відбувається реалізація звітів та можливість їх запису в формати JSON. 

	- звіт по всім працівникам

	- звіт по всім авто

	- звіть по всім продажам

	- звіт по продажу за весь час/день/конкретний період

	- звіт по продажу кожного працівника окремо

	- звіт по найбільш продаваному авто за період

	- звіт по найкращому працівнику по продажам за період

	- звіт по загальній сумі доходу

6. additional_functions.py
 В цьому файлі створені додаткові функції для орбробки даних введених користувачем з клавіатури, для того щоб програма могла коректно опрацьовувати їх та реагувити на всі можливі помилки такі як:
	- перевірка на введення дати у вказаному форматі

	- перевірка на збереження інформації у файл

	- запис у файл

	- коректність вибору користувача (в заздалегідь вказаних межах)

	- коректність введення ID співробітника/авто

	- перевірка на введення пустого значення

	- перевірка на введення слова в якому не можуть міститись цифри

	- перевірка на правильність написання електронної адреси

	- перевірка на правильність введення мобільного номеру телефону

	- перевірка на введення числового значення

7. main.py

Мозок програми, тут відбувається реалізація всіх функцій які були описані вище.
Тут користувачеві надається можливість фізично перейти в будь який розділ. Вибрати, переглянути, або змінити будь які дані які дозволені програмою.

8. .env
Зберігаються дані для підключення до БД
